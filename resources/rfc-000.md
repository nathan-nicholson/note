# RFC: Note CLI Tool

## Status
**Proposed** - v0 Specification

## Summary
A lightweight CLI tool for capturing quick notes and managing todos with project-based organization. Built in Go with local SQLite storage, designed for rapid daily capture with tagging, importance flags, and date-based organization.

---

## Motivation
Knowledge workers need a fast, keyboard-driven way to capture thoughts and tasks throughout the day without context-switching to heavy applications. This tool provides:
- Instant note capture from the terminal
- Project-based organization for focused work
- Automatic activity logging for todo and project changes
- Simple, date-based retrieval

---

## Design

### Core Concepts

**Notes:** Timestamped text entries with optional tags and importance flags.

**Todos:** Actionable items with completion status, optional due dates, and tags.

**Projects:** Organizational containers with automatic tagging. Always one active project. The `home` project is the default and always exists.

**Activity Notes:** Auto-generated notes that log todo and project lifecycle events (create, update, complete, delete, close, reopen, activate, deactivate).

---

## Command Structure

### Binary Name
`note`

### Note Commands
```bash
note <content> [--tag TAG]... [--important]
note list [--start DATE] [--end DATE] [--tag TAG]... [--important] [--show-ids]
note edit <id> [--content CONTENT] [--tag TAG]... [--important]
note delete <id>
note show <id>
note tags
```

### Todo Commands
```bash
note todo <content> [--tag TAG]... [--due DATE]
note todo list [--complete] [--incomplete] [--tag TAG]... [--overdue] [--show-ids]
note todo edit <id> [--content CONTENT] [--tag TAG]... [--due DATE]
note todo delete <id>
note todo show <id>
note todo complete <id>
note todo uncomplete <id>
```

### Project Commands
```bash
note project <project-name>                      # Switch to project
note project create <project-name> [--tag TAG]...
note project close <project-name>
note project reopen <project-name>
note project list [--all]
note project status [--all]
note project show <project-name>
note project edit <project-name> [--tag TAG]...
note project delete <project-name>
```

## Data Model

### Schema

```sql
-- Notes table
CREATE TABLE notes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    content TEXT NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    is_important BOOLEAN NOT NULL DEFAULT 0
);

-- Todos table
CREATE TABLE todos (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    content TEXT NOT NULL,
    is_complete BOOLEAN NOT NULL DEFAULT 0,
    due_date DATE,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- Tags table
CREATE TABLE tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE
);

-- Note-Tag junction
CREATE TABLE note_tags (
    note_id INTEGER NOT NULL,
    tag_id INTEGER NOT NULL,
    FOREIGN KEY (note_id) REFERENCES notes(id) ON DELETE CASCADE,
    FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE,
    PRIMARY KEY (note_id, tag_id)
);

-- Todo-Tag junction
CREATE TABLE todo_tags (
    todo_id INTEGER NOT NULL,
    tag_id INTEGER NOT NULL,
    FOREIGN KEY (todo_id) REFERENCES todos(id) ON DELETE CASCADE,
    FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE,
    PRIMARY KEY (todo_id, tag_id)
);

-- Projects table
CREATE TABLE projects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    first_activated_at TIMESTAMP,
    last_activity_at TIMESTAMP,
    closed_at TIMESTAMP,
    is_closed BOOLEAN DEFAULT 0
);

-- Project-Tag junction
CREATE TABLE project_tags (
    project_id INTEGER NOT NULL,
    tag_id INTEGER NOT NULL,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
    FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE,
    PRIMARY KEY (project_id, tag_id)
);

-- Active project (single row)
CREATE TABLE active_project (
    project_id INTEGER NOT NULL UNIQUE,
    activated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);
```

### Constraints
- Project names must be kebab-case and unique
- Project names cannot be reserved command names (create, close, reopen, list, status, show, edit, delete)
- Always exactly one active project at a time
- The `home` project always exists and cannot be deleted
- Tags are case-sensitive and reusable across notes, todos, and projects
- Projects can only be closed if all associated todos are complete

### First Run Initialization
```sql
-- Create default home project
INSERT INTO projects (name, created_at, first_activated_at, is_closed)
VALUES ('home', CURRENT_TIMESTAMP, CURRENT_TIMESTAMP, 0);

-- Set as active project
INSERT INTO active_project (project_id, activated_at)
SELECT id, CURRENT_TIMESTAMP FROM projects WHERE name = 'home';
```

## Features

### 1. Quick Note Capture
```bash
note "Discussed list format with team"
# Creates note with current timestamp, tagged with active project

note "Important design decision" --tag architecture --important
# Creates note tagged #architecture and current project name, marked important
```

**Output:** Silent success (or brief confirmation)

---

### 2. Date-Based Note Listing

**Default behavior:** Shows today's notes only

```bash
note list
```

**Output:**
```
2025-11-23
  09:15 AM  Remember to call dentist #personal #home
  02:30 PM  [!] Review PR before EOD #work #urgent #home
  04:45 PM  Idea for blog post about Go CLI tools #writing #home
```

**Date range queries:**
```bash
note list --start 2025-11-20              # From Nov 20 to today
note list --start 2025-11-20 --end 2025-11-22  # Specific range
```

**Filtering:**
```bash
note list --tag work                      # Only #work notes
note list --important                     # Only important notes
note list --show-ids                      # Show IDs for editing/deleting
```

**Format:**
- Date headers group notes by day
- Time shown in 12-hour format (09:15 AM)
- `[!]` prefix for important notes
- Tags displayed inline with `#` prefix
- Most recent notes first within each day

---

### 3. Todo Management

**Create todos:**
```bash
note todo "Review performance metrics" --tag work
# Automatically tagged with current project name

note todo "File taxes" --due 2025-12-31 --tag finance
# Automatically tagged with current project name
```

**List todos:**
```bash
note todo list
```

**Output:**
```
OVERDUE
  [ ] [#12] 2025-11-20  File tax documents #finance #home

TODAY (2025-11-23)
  [X] [#15] Review morning emails #work #home
  [ ] [#16] Call insurance company (due today) #personal #home

UPCOMING
  [ ] [#17] 2025-11-25  Prepare presentation #work #home
  [ ] [#18] 2025-11-28  Birthday gift for mom #personal #home

NO DUE DATE
  [ ] [#19] Learn about Docker networking #learning #home
```

**Status-based grouping:**
- OVERDUE: Past due date, incomplete
- TODAY: Due today (both complete and incomplete)
- UPCOMING: Future due dates
- NO DUE DATE: Todos without due dates

**Sort order within groups:**
- By due date (if exists)
- Then by creation date

**IDs always shown:**
- Todo IDs are always displayed in format `[#42]`
- No `--show-ids` flag needed for todos

**Complete/uncomplete:**
```bash
note todo complete 42
note todo uncomplete 42
```

### 4. Activity Notes

Auto-generated notes track todo and project lifecycle events.

#### Todo Activity Notes

**Format:**
```
Created todo: <content> [(due: <date>)] [#tags] #todo #create
Updated todo: <changes> #todo #update
Completed todo: <content> [(due: <date>)] [#tags] #todo #complete
Deleted todo: <content> [(due: <date>)] [#tags] #todo #delete
```

**Examples:**
```
2025-11-23
  09:15 AM  Created todo: Make breakfast #personal #home #todo #create
  10:30 AM  Completed todo: Make breakfast #personal #home #todo #complete
  02:15 PM  Updated todo: Updated content to "Make brunch", due date to 2025-11-23 #todo #update
  03:45 PM  Deleted todo: Make brunch (due: 2025-11-23) #personal #home #todo #delete
```

**Automatic tags:**
- `#todo` - Identifies todo activity note
- `#create`, `#update`, `#complete`, `#delete` - Action type

**Update notes consolidate multiple changes:**
```
Updated todo: Updated content to "New content", due date to 2025-11-25, tags to #work #urgent
```

#### Project Activity Notes

**Format:**
```
Created project: <project-name> [#tag1 #tag2...] #project #create
Activated project: <project-name> #project #activate
Deactivated project: <project-name> #project #deactivate
Updated project: <project-name> - <changes> #project #update
Closed project: <project-name> #project #close
Reopened project: <project-name> #project #reopen
Deleted project: <project-name> [#tag1 #tag2...] #project #delete
```

**Examples:**
```
2025-11-23
  09:00 AM  Created project: work #professional #project #create
  09:01 AM  Activated project: work #project #activate
  10:30 AM  Updated project: work - Updated tags to #professional #fulltime #project #update
  04:30 PM  Deactivated project: work #project #deactivate
  04:30 PM  Closed project: work #project #close
```

**Automatic tags:**
- `#project` - Identifies project activity note
- `#create`, `#activate`, `#deactivate`, `#update`, `#close`, `#reopen`, `#delete` - Action type

**Activity notes are regular notes:**
- Appear in `note list` with other notes
- Can be filtered: `note list --tag complete` or `note list --tag project`
- No special database fields

### 5. Project-Based Organization

#### Default Project

The `home` project is:
- Created automatically on first run
- Set as active by default
- Always exists (cannot be deleted)
- Can be closed and reopened like any other project
- Used as fallback when deleting active projects

**First run generates:**
```
Created project: home #project #create
Activated project: home #project #activate
```

#### Create Projects

```bash
note project create work --tag professional
note project create blog-redesign --tag personal --tag web
```

**Activity note generated:**
```
Created project: work #professional #project #create
```

#### Switch Projects

```bash
note project work
```

**Activity notes generated:**
```
Deactivated project: home #project #deactivate
Activated project: work #project #activate
```

**Auto-tagging behavior:**
While a project is active, all new notes and todos are automatically tagged with the project name:
```bash
# Active project: work
note "Finished sprint planning"
# Automatically tagged: #work

note todo "Review PRs" --due today
# Automatically tagged: #work
```

#### Project Status

```bash
note project status
```

**Output:**
```
Project: work
Tags: #professional
Started: 2025-11-20
Last Updated: 2025-11-23
Tasks: 5/12 complete (42%)

Incomplete Tasks:
  [ ] [#23] Install new fixtures (due: 2025-11-25) #work
  [ ] [#24] Order cabinet hardware (due: 2025-11-28) #work
  [ ] [#25] Paint living room #work
  [ ] [#26] Research tile options #work
```

**With `--all` flag:**
```bash
note project status --all
```

**Output includes completed tasks section:**
```
Project: work
Tags: #professional
Started: 2025-11-20
Last Updated: 2025-11-23
Tasks: 5/12 complete (42%)

Incomplete Tasks:
  [ ] [#23] Install new fixtures (due: 2025-11-25) #work
  [ ] [#24] Order cabinet hardware (due: 2025-11-28) #work
  [ ] [#25] Paint living room #work
  [ ] [#26] Research tile options #work

Completed Tasks:
  [X] [#18] Buy paint samples #work
  [X] [#19] Measure windows (was due: 2025-11-21) #work
  [X] [#20] Remove old fixtures #work
  [X] [#21] Clean walls #work
  [X] [#22] Order supplies (was due: 2025-11-22) #work
```

**Field definitions:**
- **Started:** When project was first set as active
- **Last Updated:** Last time a note or todo was added with this project tag
- **Tasks:** Count of all todos tagged with project name
- **Incomplete Tasks:** Sorted by due date (if exists), then created date
- **Completed Tasks:** Sorted by completion date (most recently completed first)

#### List Projects

```bash
note project list
```

**Output (default - open projects only):**
```
ACTIVE
  * work #professional

OPEN
  blog-redesign #personal #web
  home
  side-project #learning
```

**With `--all` flag:**
```bash
note project list --all
```

**Output:**
```
ACTIVE
  * work #professional

OPEN
  blog-redesign #personal #web
  home
  side-project #learning

CLOSED
  old-prototype #archived
  summer-2024 #personal
  website-v1 #personal #web
```

**Section definitions:**
- **ACTIVE:** Currently active project (exactly 1), marked with `*`
- **OPEN:** Non-closed projects that are not active, sorted alphabetically
- **CLOSED:** Closed projects, sorted by closed date (most recent first), only shown with `--all`

#### Close Project

```bash
note project close work
```

**Requirements:**
- All todos tagged with project name must be completed
- If project is `home` and it's the only open project, cannot close
- If incomplete todos exist, shows error with list

**Output (with incomplete todos):**
```
Error: Cannot close project 'work' - 4 incomplete todos remaining

Incomplete Tasks:
  [ ] [#23] Install new fixtures (due: 2025-11-25) #work
  [ ] [#24] Order cabinet hardware (due: 2025-11-28) #work
  [ ] [#25] Paint living room #work
  [ ] [#26] Research tile options #work

Complete all todos before closing the project.
```

**Output (success):**
```
Project 'work' closed successfully.
```

**Behavior:**
- If project is active, automatically deactivates and switches to another open project
- Marks project as closed with timestamp
- Generates activity notes for deactivation and closure

**Activity notes generated (when closing active project):**
```
Deactivated project: work #project #deactivate
Activated project: home #project #activate
Closed project: work #project #close
```

#### Reopen Project

```bash
note project reopen work
```

**Behavior:**
- Removes closed status
- Generates activity note
- Project can now be set as active again

**Activity note generated:**
```
Reopened project: work #project #reopen
```

#### Show Project Details

```bash
note project show work
```

**Output (open project):**
```
Project: work
Tags: #professional
Status: Open
Created: 2025-11-20
First Activated: 2025-11-20
Last Activity: 2025-11-23
Tasks: 5/12 complete (42%)
```

**Output (closed project):**
```
Project: work
Tags: #professional
Status: Closed
Created: 2025-11-20
First Activated: 2025-11-20
Closed: 2025-11-23
Tasks: 12/12 complete (100%)
```

#### Edit Project Tags

```bash
note project edit work --tag professional --tag fulltime
```

**Activity note generated:**
```
Updated project: work - Updated tags to #professional #fulltime #project #update
```

#### Delete Project

```bash
note project delete old-project
```

**Constraints:**
- Cannot delete `home` project
- If deleting active project, automatically switches to `home` first

**Output (deleting home):**
```
Error: Cannot delete the 'home' project. It is the default project and must always exist.
```

**Behavior (deleting active project):**
1. Deactivate current project
2. Switch to `home`
3. Delete the project

**Activity notes generated:**
```
Deactivated project: old-project #project #deactivate
Activated project: home #project #activate
Deleted project: old-project #archived #project #delete
```

### 6. Date Input Formats

**ISO Format:**
```
2025-11-23
2025-12-31
```

**Natural Language (Limited):**
- `today` - Current date
- `tomorrow` - Next day
- `end-of-week` - Next Friday
- `end-of-month` - Last day of current month
- `next-week` - 7 days from today
- `next-month` - Same day next month

**Examples:**
```bash
note todo "Submit report" --due tomorrow
note todo "Quarterly review" --due end-of-month
note todo "Follow up meeting" --due next-week
```

### 7. Tagging System

**Tag format:**
- Lowercase, alphanumeric with hyphens
- Displayed with `#` prefix in output
- Case-sensitive storage

**Multiple tags:**
```bash
note "Meeting notes" --tag work --tag standup --tag q4
note todo "Review" --tag work --tag urgent
```

**List all tags:**
```bash
note tags
```

**Output:**
```
personal (45 uses)
work (123 uses)
urgent (12 uses)
home (156 uses)
todo (89 uses)
project (45 uses)
create (43 uses)
complete (20 uses)
close (5 uses)
```

**Tag filtering:**
```bash
note list --tag work --tag urgent    # AND logic (must have both)
note todo list --tag personal
```

**Automatic activity tags:**

*Todo activity:*
- `#todo` - Todo activity identifier
- `#create` - Todo creation
- `#update` - Todo updates
- `#complete` - Todo completion
- `#delete` - Todo deletion

*Project activity:*
- `#project` - Project activity identifier
- `#create` - Project creation
- `#activate` - Project activation
- `#deactivate` - Project deactivation
- `#update` - Project updates
- `#close` - Project closure
- `#reopen` - Project reopening
- `#delete` - Project deletion

*Project name tags (auto-applied):*
- Automatically added to notes/todos when project is active
- Example: `#home`, `#work`, `#blog-redesign`

### 8. Edit and Delete Operations

**Show IDs:**
```bash
note list --show-ids
```

**Output:**
```
2025-11-23
  [#42] 09:15 AM  Remember to call dentist #personal #home
  [#43] 02:30 PM  [!] Review PR before EOD #work #urgent #home
```

**Edit note:**
```bash
note edit 42 --content "New content"
note edit 42 --tag newTag
note edit 42 --important
```

**Edit todo:**
```bash
note todo edit 42 --content "Updated task"
note todo edit 42 --due 2025-12-01
note todo edit 42 --tag work --tag urgent
```

**Delete:**
```bash
note delete 42
note todo delete 42
```

**Behavior:**
- Edits and deletes require IDs
- Note IDs are hidden by default, shown with `--show-ids`
- Todo IDs are always shown
- All edits generate activity notes for todos
- Deletes generate activity notes for todos

## Display Format Specifications

### Time Format
- 12-hour format with AM/PM
- Example: `09:15 AM`, `03:45 PM`

### Date Headers
- ISO format: `2025-11-23`
- Shown as section headers grouping notes by day

### Important Note Indicator
- `[!]` prefix before content
- Example: `[!] Review PR before EOD`

### Completion Checkbox
- `[ ]` for incomplete
- `[X]` for complete

### ID Display
- Format: `[#42]`
- Notes: Only shown with `--show-ids` flag
- Todos: Always shown
- Project status tasks: Always shown

### Tag Display
- Hash prefix: `#work`, `#personal`
- Inline at end of content
- Space-separated

### Project Status Indicators
- `*` - Currently active project (in project list)
- `(closed)` - Closed project (in project list)

## Error Handling

### User Errors

**Non-existent ID:**
```
Error: Note #999 not found
```

**Non-existent project:**
```
Error: Project 'doesnt-exist' not found. Create it with: note project create doesnt-exist
```

**Closed project:**
```
Error: Cannot set closed project 'old-project' as active. Reopen it with: note project reopen old-project
```

**Cannot close project with incomplete todos:**
```
Error: Cannot close project 'work' - 4 incomplete todos remaining

Incomplete Tasks:
  [ ] [#23] Install new fixtures (due: 2025-11-25) #work
  [ ] [#24] Order cabinet hardware (due: 2025-11-28) #work
  [ ] [#25] Paint living room #work
  [ ] [#26] Research tile options #work

Complete all todos before closing the project.
```

**Cannot close last open project:**
```
Error: Cannot close 'home' project - it is the only open project. Create or reopen another project first.
```

**Cannot delete home project:**
```
Error: Cannot delete the 'home' project. It is the default project and must always exist.
```

**Reserved project name:**
```
Error: Project name 'list' is reserved. Choose a different name.
Reserved names: create, close, reopen, list, status, show, edit, delete
```

**Invalid date format:**
```
Error: Invalid date '2025-13-45'. Use YYYY-MM-DD or: today, tomorrow, end-of-week, end-of-month, next-week, next-month
```

**Invalid project name:**
```
Error: Project name must be kebab-case (lowercase letters and hyphens only)
```

### System Errors

**Database connection failure:**
```
Error: Could not connect to database at ~/.note/notes.db
```

**Permission issues:**
```
Error: Could not create directory ~/.note (permission denied)
```
